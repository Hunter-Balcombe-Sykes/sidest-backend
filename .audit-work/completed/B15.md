# B15 — Shopify storefront token hardening

Credited manually after the fact — the work landed as two real commits but the
orchestrator never produced a B15 completion record at the time.

- **V5-004 (encryption-at-rest)**: Storefront access token moved out of the plaintext
  `provider_metadata` JSON column into a dedicated `storefront_token TEXT` column on
  `professional_integrations`, with Laravel's `'encrypted'` cast (AES-256-CBC). All read
  and write sites updated to use the new column.
- **V5-003 (reinstall revocation)**: `BrandSignupService` now revokes the old Storefront
  Access Token at Shopify via REST before swapping the admin API token, then nulls the
  local column so `CreateStorefrontAccessTokenJob` mints a fresh token against the new
  credential. 404s and transport errors are tolerated so a failed revocation never blocks
  a reinstall.
- **4-04 (ride-along)**: `IntegrationTokenEncryptionTest` (Feature/Security) verifies
  `storefront_token`, `access_token`, and `refresh_token` are persisted as ciphertext —
  reading the raw row bypassing the cast must not match the plaintext value.
