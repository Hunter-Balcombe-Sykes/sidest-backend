## Plain English

The analytics click and pageview validators were checking whether a block or site existed using an ambiguous table reference that depended on a PostgreSQL configuration setting (search_path) to resolve correctly. This meant the validation could silently fail or hit the wrong table in different environments. The fix makes the table reference explicit so it always points to the right place, regardless of server configuration.

## Technical Summary

**Files changed:**
- `app/Http/Requests/Api/PublicSite/Analytics/ClickRequest.php` — `block_id` and `site_id` exists rules changed from `'blocks'`/`'sites'` to `'pgsql.site.blocks'`/`'pgsql.site.sites'`
- `app/Http/Requests/Api/PublicSite/Analytics/PageviewRequest.php` — `site_id` exists rule changed from `'sites'` to `'pgsql.site.sites'`
- `tests/Feature/Security/TenantIsolation/PublicAnalyticsIdorTest.php` — removed shadow `public.sites` table creation and stale inserts that were workarounds for the unqualified rule

**New behavior:** `Rule::exists('pgsql.site.sites', 'id')` is parsed by Laravel as connection=`pgsql`, table=`site.sites`. The query builder generates `SELECT COUNT(*) FROM "site"."sites"` on PostgreSQL and correctly accesses the attached `site` database in the SQLite test environment.

## Decisions Made

**Why not `Rule::exists('site.sites', 'id')`?** Laravel's `DatabasePresenceVerifier` treats the part before the first dot as a *connection name*, not a schema name. `Rule::exists('site.sites', 'id')` tries to open a database connection named `'site'`, which doesn't exist → `InvalidArgumentException`. The correct pattern for schema-qualified tables is `'connection.schema.table'` (split on first dot only gives connection + schema.table).

**Why also fix `sites` in ClickRequest?** The audit only called out `blocks`, but `site_id` validation in the same file had the same unqualified reference. Fixed both for consistency.

**Test cleanup:** The `PublicAnalyticsIdorTest` had a shadow `public.sites` table setup specifically to work around the unqualified `sites` rule. With the rule now pointing to `pgsql.site.sites`, `createBrandTenant` already inserts the site into the real `site.sites` table — the shadow table and its inserts were dead code.
