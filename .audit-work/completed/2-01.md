# #2-01 — Brand-partner enrichment trusts affiliate-controlled JSON for brand_professional_id lookup

## Plain English

When an affiliate's public site loads, the system used to read a brand ID straight out of the affiliate's own settings (which they can edit) and immediately fetch that brand's logos and images to display. An affiliate could put any brand's ID in there — even a brand that never partnered with them — and their images would show up. The fix checks the actual partnership database table first. If there's no confirmed partnership record linking that affiliate to that brand, the enrichment is skipped entirely and a warning is logged.

## Technical Summary

**File changed:** `app/Services/Cache/SiteCacheService.php` — `applyBrandImageFallbacks()` method.

**Before:** `$brandId` was read directly from `$payload['site']['settings']['brand_partner']['professional_id']` (affiliate-controlled JSON) and used immediately to query `Site` for brand media.

**After:**
1. `$affiliateId` is resolved from `$payload['professional']['id']` (the authenticated site owner).
2. The JSON value is kept as `$claimedBrandId` — a hint only.
3. `BrandPartnerLink::where('affiliate_professional_id', $affiliateId)->where('brand_professional_id', $claimedBrandId)->first()` verifies the consent record exists.
4. If no link is found, `Log::warning` fires with both `affiliate_id` and `brand_id`, and the method returns the payload unchanged.
5. `$brandId` is resolved from `$link->brand_professional_id` (the DB record), never from the JSON.

## Decisions Made

- **Use `->first()` not `->exists()`**: The audit item said `exists()` for the check, but retrieving the full record with `first()` lets us resolve `brand_professional_id` from the link in one query instead of two. Functionally equivalent for the security guarantee; one fewer DB round-trip.
- **Log level `warning` not `info`**: A missing link when JSON advertises a brand partner is an anomaly worth alerting on (misconfiguration or attempted abuse), not routine noise.
- **Kept the JSON `is_array` / `empty` guard intact**: It remains a cheap short-circuit before hitting the DB for the common case where no brand partner is configured at all.

## Notes

`BrandPartnerLink` was already imported at the top of the file (line 6), so no additional use statement was needed. The `brandPartnerEnrichmentCache` property on the class is used elsewhere (typography hydration path) and was unaffected by this change.
