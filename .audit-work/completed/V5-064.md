## Plain English

Phone number fields across the app were each enforcing their own maximum length — some said 30 characters, one said 40, another said 64, and some had no length limit at all. This meant a phone number accepted on one form might be rejected on another. The fix adds a single shared rule to the base request class so every form now enforces the same 50-character cap, and changing it in the future only requires editing one place.

## Technical Summary

**Files changed:** 11 (1 base class + 10 Form Requests)

- `BaseFormRequest`: added `phoneRule(): array` returning `['string', 'max:50']`
- 10 Form Requests updated to spread `...$this->phoneRule()` in place of inline `'string', 'max:XX'`:
  - `BootstrapRequest` — was `max:40` → now `max:50`
  - `PublicEnquiryRequest` — was `max:30` → now `max:50`
  - `UpsertGoogleBusinessProfileRequest` — was `max:64` → now `max:50`
  - `PublicWaitlistSignupRequest` — had no `max:` (regex only) → now `max:50` prepended before regex
  - `StoreCustomerRequest`, `PublicCustomerLeadRequest`, `UpdateCustomerRequest`, `StaffUpdateCustomerRequest`, `UpdateProfessionalRequest`, `StaffUpdateProfessionalRequest` — already `max:50` but now reference the canonical method; `public_contact_number` in the last two also updated

**Contract:** callers control presence modifiers (`required`/`nullable`/`sometimes`) independently; `phoneRule()` contributes only `['string', 'max:50']`.

## Decisions Made

- `max:50` chosen as canonical value — used by 6 of 10 existing callsites and comfortably covers E.164 + any reasonable formatted representation (longest real-world formatted number with extension ≈ 26 chars).
- `UpsertGoogleBusinessProfileRequest` was at `max:64` with no `normalizePhones()` call (raw trimmed input). max:50 is still safe — even fully-formatted international numbers with extensions fit well within 50 chars.
- `PublicWaitlistSignupRequest`'s `regex:/^\+?[0-9]{7,20}$/` was preserved; its own `normalizePhone()` strips non-digits before validation so the regex is effectively stricter than max:50 anyway.
