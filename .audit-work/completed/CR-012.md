## Plain English

When a commission payout is created, the system sets a "void by" deadline on it. Previously, that deadline was calculated using the app server's clock, while the payout record's creation timestamp was set by the database's internal clock. These two clocks are usually the same, but can drift by milliseconds to seconds under load. The fix makes both timestamps come from the same database clock, keeping them consistent.

## Technical Summary

**Files changed:** `app/Services/Stripe/CommissionPayoutService.php` (line 219)

**Change:** Replaced `now()->addDays($graceDays)` with `DB::raw("NOW() + INTERVAL '{$graceDays} days'")`.

**New behavior:** `void_at` is now computed inside PostgreSQL using the transaction snapshot's `NOW()`, the same clock that stamps `created_at`. This eliminates any app-server vs. DB clock divergence on the grace window calculation.

**Contract:** `$graceDays` is an integer sourced from config, so the interpolation into `DB::raw()` carries no SQL injection risk.

## Notes

The drift is harmless at the current grace window of days, but the fix is the correct pattern regardless — consistency between related timestamps should not depend on how fast the app server is.
