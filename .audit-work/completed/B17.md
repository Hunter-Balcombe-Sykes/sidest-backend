# B17 ã Square/Fresha job retry hygiene

## Plain English

All four Square and Fresha background jobs now have explicit retry settings: if a job fails (e.g. the external API is down), it will retry up to 3 times, waiting 30 seconds after the first failure and 60 seconds after the second. When a job exhausts all retries, it now logs a warning with enough context (which provider, which record, what went wrong) so you can see the failure in Nightwatch without digging through raw queue logs. Previously only the Square catalog sync job had this failure logging; the other three were silent.

## Technical Summary

**Files changed:** `app/Jobs/Square/SyncSquareCatalogDeltaJob.php`, `app/Jobs/Square/PushServiceToSquareJob.php`, `app/Jobs/Fresha/SyncFreshaCatalogDeltaJob.php`, `app/Jobs/Fresha/PushServiceToFreshaJob.php`.

**Retry config added to all four:** `public int $tries = 3` and `public array $backoff = [30, 60]`. These public properties override Laravel's queue worker defaults on a per-job basis.

**`failed(\Throwable $e)` added to three jobs:** `SyncFreshaCatalogDeltaJob` (mirrors `SyncSquareCatalogDeltaJob` exactly, swapping `merchant_id` ã `business_id`), `PushServiceToSquareJob` (logs `service_id` + `action`), and `PushServiceToFreshaJob` (same shape as Square push). `Log::warning` import added to the three files that lacked it. `SyncSquareCatalogDeltaJob` already had `failed()` ã only `$tries`/`$backoff` were added.

## Decisions Made

- **`$tries = 3`**: Standard for external API jobs ã enough attempts to ride out transient failures without burning the queue.
- **`$backoff = [30, 60]`**: 30s after attempt 1, 60s after attempt 2; fits rate-limited external APIs (Square/Fresha) and aligns with the Retry-After handling already in the API clients.
- **Added `failed()` to `PushService*` jobs even though #5-03 only named `SyncFreshaCatalogDeltaJob`**: The bundle theme is parity; leaving the push jobs without failure logging would be an obvious gap. Mechanical and safe.

## Notes

No existing tests cover job retry behavior directly ã the changes are pure property additions and a new lifecycle hook, so no test changes were needed. All 1207 tests pass.
