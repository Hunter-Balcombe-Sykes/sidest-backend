# #1-06 ā€” `BrandAccessService::isBrandProfessional()` is a type-check living next to capability methods

## Plain English
The `BrandAccessService` had a helper method that just answered "is this account a brand?" ā€” but that kind of identity question belongs on the account object itself, not in a service that handles permissions. The Professional model already had an `isBrand()` method that did the exact same thing. We deleted the duplicate and updated the two places that called it to use the model's own method directly.

## Technical Summary
- Removed `BrandAccessService::isBrandProfessional(Professional): bool` from `app/Services/Store/BrandAccessService.php` (was using `strtolower`; superseded by the model's `isBrand()` which uses `mb_strtolower` ā€” semantically identical for the ASCII string 'brand').
- Internal call in `resolvedBrandAccess()` replaced with `$professional->isBrand()`.
- External call in `ShopifyIntegrationController::resolveTargetBrandProfessionalId()` (line 66) replaced with `$professional->isBrand()`.
- `BrandAccessService` was the only reason the controller depended on it; removed the constructor injection and `use` import from `ShopifyIntegrationController`.
- Updated docblock comment in `BrandResourcePolicy` to reference `isBrand` instead of `isBrandProfessional`.
- Updated `tests/Unit/Http/ShopifyResolveShopSsrfTest.php` to drop the now-removed `BrandAccessService` mock from controller instantiation.

## Decisions Made
- **Use existing `isBrand()` rather than creating a new method**: `Professional::isBrand()` already exists and is semantically identical ā€” creating a duplicate would be the same problem in a different place.
- **Remove BrandAccessService from controller constructor**: It was the sole remaining use in the controller; removing it keeps dependency injection honest and prevents future confusion about why it's there.

## Notes
The `strtolower` vs `mb_strtolower` discrepancy between the deleted method and `Professional::isBrand()` is a non-issue ā€” the comparison target 'brand' is ASCII-only.
