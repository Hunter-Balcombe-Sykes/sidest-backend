## Plain English

Staff broadcast emails include an unsubscribe link at the bottom. The code that builds those emails was relying on a quiet Laravel convention to pass data to the email template, but the unsubscribe URL was never explicitly handed to the template — a subtle gap that could silently break if the mail class is ever updated. The fix makes it explicit: the notification content and unsubscribe URL are now clearly declared as inputs to the template.

## Technical Summary

**File changed:** `app/Mail/StaffBroadcastMail.php`

`build()` previously called `->view('emails.staff_broadcast')` with no explicit data. Laravel's `buildViewData()` does auto-pass public properties, but `$unsubscribeUrl` (used on line 17 of the Blade template) was never guaranteed to arrive if the class is ever migrated to the newer `envelope()`/`content()` API.

Changed to `->view('emails.staff_broadcast', ['notification' => ..., 'unsubscribeUrl' => ...])` so the view's data contract is explicit and self-documenting.

## Decisions Made

Passed both `$notification` and `$unsubscribeUrl` explicitly (not just `$unsubscribeUrl`) so the full view contract is visible at a glance, even though `$notification` was previously safe via auto-passing.
