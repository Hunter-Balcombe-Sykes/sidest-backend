## Plain English

Affiliates who haven't connected their Stripe account receive two reminder emails — one with 10 days left and one with 2 days left. A scheduling bug meant the system could send each reminder up to three days in a row instead of just once. This fix narrows each sending window to a single calendar day so each reminder fires exactly once per affiliate.

## Technical Summary

**File changed:** `app/Services/Stripe/CommissionVoidService.php:135-146`

`sendSignupWarnings()` uses `whereBetween` on `stripe_grace_period_ends_at` to find affiliates in each warning window. The windows were 3 days wide (`addDays(9)..addDays(11)` and `addDays(1)..addDays(3)`), which meant a daily cron could match the same row on up to three successive days.

Changed both windows to single-day spans:
- `day20`: `addDays(10)->startOfDay()` .. `addDays(10)->endOfDay()`
- `day28`: `addDays(2)->startOfDay()` .. `addDays(2)->endOfDay()`

The center points (`+10` and `+2` days) align with the "10 days left" and "2 days left" copy in the notification body/title. No schema changes, no new dependencies.

## Decisions Made

Chose `addDays(10)` (not 9 or 11) for the day-20 window center because the notification title reads "10 days left," and the window should match exactly when `stripe_grace_period_ends_at` is 10 days out. Same logic applies to `addDays(2)` for the day-28 window.
