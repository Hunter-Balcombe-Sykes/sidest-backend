# #CR-015 ŌĆö `role` query param trusted without validation in StripeConnectController::payouts

## Plain English
The payouts endpoint lets a professional ask "show me payouts where I'm the brand" or "show me payouts where I'm the affiliate" by passing a `role` query parameter. Previously any value could be sent (or the field could be omitted and it would silently default to affiliate). Now the endpoint rejects requests that don't include `role` with exactly "brand" or "affiliate" ŌĆö callers get a clear 422 validation error instead of silent misbehavior.

## Technical Summary
Created `app/Http/Requests/Api/Professional/Stripe/PayoutsRequest.php` extending `BaseFormRequest` with `'role' => ['required', Rule::in(['brand', 'affiliate'])]`. Updated `StripeConnectController::payouts()` to accept `PayoutsRequest` instead of `Request`, reading the param via `$request->input('role')` rather than `$request->validated()` (the latter requires the HTTP kernel's `validateResolved` lifecycle which doesn't run in direct-invocation tests). Updated two test files ŌĆö `StripeConnectPayoutsControllerTest.php` (changed helper to use `PayoutsRequest::create()` with default `role=affiliate`) and `CommissionsIsolationTest.php` (replaced `tenantRequestAs` call with a direct `PayoutsRequest::create()` + attribute set).

## Decisions Made
- `$request->input('role')` over `$request->validated()['role']`: the Form Request validation gate ensures the value is valid before the controller runs in production, so `input()` is semantically equivalent there while avoiding the null-validator crash in unit tests.
- Default `role=affiliate` baked into `makePayoutsRequest` helper: keeps existing test intent ("by default = affiliate") without requiring every call site to be updated.

## Notes
`CommissionsIsolationTest.php` was a previously unknown second call site. The `void_at` column referenced in the response map is absent from both test table DDLs but Eloquent returns null for missing attributes, so those tests were already passing without it ŌĆö no DDL change needed.
