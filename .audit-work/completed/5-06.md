# #5-06 — Fresha booking methods are dead code

## Plain English

Fresha, the salon booking platform, doesn't let third-party apps create or manage bookings via their API. Four methods had been written as if they would — searching availability, creating bookings, cancelling them, and creating customers. None were called anywhere. Removed all four stubs and updated the class comment to reflect that this client is only used for service sync.

## Technical Summary

Removed four public methods from `app/Services/Fresha/FreshaApiClient.php`:
- `searchAvailability(Professional, array): array`
- `createBooking(Professional, array): array`
- `cancelBooking(Professional, string): array`
- `createCustomer(Professional, array): array`

Also updated the class-level inline comment from "services, bookings, and availability" to "service sync only". All 1464 tests pass.

## Decisions Made

- Removed all four methods together: they form a cohesive "phantom booking API" group.
- Updated the class comment: it explicitly mentioned bookings, making it dead documentation.

## Notes

`FreshaApiClient` is still a real, used class — `FreshaServiceSyncService` calls `fetchServices`, `createService`, `updateService`, `deleteService`, and `retrieveService`. Only the booking-side stubs were removed.
