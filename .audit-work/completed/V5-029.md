## Plain English

When a commission payout is created, a "void by" deadline is stamped on it so that if the affiliate never activates their Stripe account, the payout is automatically cancelled. That deadline was calculated using a config value (`grace_period_days`) that had no guard rails — if someone accidentally set it to 0 or a negative number, every new payout would be voided immediately. This fix ensures the value is always between 1 and 365 days, no matter what's in the config.

## Technical Summary

**File changed:** `app/Services/Stripe/CommissionPayoutService.php`

- Added `private int $gracePeriodDays` property.
- In `__construct`, read `sidest.store.grace_period_days` (default 60) and clamp to `[1, 365]` via `max(1, min(365, ...))` — same defensive pattern already used for `systemHoldDays`.
- Replaced the inline `config(...)` call at the payout-creation site (line 207) with `$this->gracePeriodDays`. The `void_at` SQL expression `NOW() + INTERVAL '{$this->gracePeriodDays} days'` is unchanged in structure.

**New behavior:** misconfigured values (0, negative, or >365) are silently clamped rather than propagated into the DB. The default of 60 is unaffected.
