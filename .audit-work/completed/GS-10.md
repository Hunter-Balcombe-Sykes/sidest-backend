# #GS-10 â€” Wire phpredis serializer (igbinary) + LZ4 compression when payload size warrants

## Plain English

We've wired up a more efficient storage format for Redis cached data â€” one that uses roughly 3Ă— less memory and sends less data over the network. It's currently switched off by default. Before turning it on, someone needs to verify the server has the right Redis extensions installed (igbinary and LZ4), and then flush the cache when deploying so old-format entries don't cause errors. The new `REDIS_IGBINARY=true` env var is the on/off switch.

## Technical Summary

- **`config/database.php`**: Added an `'options'` array to the `cache` Redis connection. When `REDIS_IGBINARY=true`, sets `serializer => Redis::SERIALIZER_IGBINARY` and `compression => Redis::COMPRESSION_LZ4` via phpredis's option API. `defined()` guards on both constants prevent fatal errors if phpredis was compiled without igbinary/LZ4 support. `array_filter` keeps the options array empty (no overrides) when the flag is off.
- **`.env.example`**: Added `REDIS_IGBINARY=false` with inline documentation covering the two activation prerequisites and the payload-size trigger threshold.
- Applied only to the `cache` connection (not `session`, `queue`, `default`, or `cache_locks`) â€” session and queue connections serialize their own payloads independently and must not get a second serialization layer.

## Decisions Made

- **Cache connection only, not global**: Applying igbinary globally via the top-level `'options'` block would break the session connection (double serialization) and the queue connection (Horizon expects plain JSON strings). Targeted per-connection options is the safe scope.
- **`array_filter` over explicit null checks**: Cleaner than conditionally building the array with `if` statements; safe because the target constants (`SERIALIZER_IGBINARY` = 2, `COMPRESSION_LZ4` = 3) are non-zero and won't be filtered out when set.
- **Single env var `REDIS_IGBINARY` for both serializer and compression**: They're always activated together â€” enabling compression without igbinary (or vice versa) has no practical use case here. One knob is simpler to operate.
- **Did not add ZSTD as an alternative**: The audit item specifies LZ4; ZSTD offers better ratios but more CPU. Left as a future upgrade path if memory pressure warrants it.

## Notes

The `defined()` guard pattern (e.g. `defined('Redis::SERIALIZER_IGBINARY')`) is the correct way to check phpredis class constants in PHP 8.2 â€” it returns false if the constant doesn't exist on the class, preventing the fatal error that would occur from directly referencing an undefined constant.

Activation checklist for Josh at deploy time:
1. Run `phpinfo()` on the production server and confirm `igbinary support = enabled` and `lz4 compression = enabled` under the Redis section.
2. Flush the Redis cache (`php artisan cache:clear` or `redis-cli FLUSHDB` on cache DB).
3. Set `REDIS_IGBINARY=true` in the production environment.
