## Plain English

When Shopify sends a webhook about a paid order, each line item includes a product ID number. Previously that number was used directly to build a Shopify internal identifier without checking that it was actually a number. A tampered webhook could have injected unexpected characters into that identifier. Now the code confirms the value is digits-only before using it, and silently ignores any line item that fails that check.

## Technical Summary

**File changed:** `app/Jobs/Shopify/ProcessShopifyOrderWebhookJob.php` (lines 100–103)

**Change:** Added `preg_match('/^\d+$/', $productId)` guard before constructing the `gid://shopify/Product/{$productId}` string. Non-numeric values are skipped, consistent with the existing `is_array($li)` guard above it.

**New behavior:** Only strictly numeric `product_id` values from the webhook payload are ever embedded in a GID. Any other value (empty, alphanumeric, path-like) is silently dropped — the order processing continues for other line items.

**Contracts unchanged:** No public API surface changed; no job signature change; no DB schema change.
