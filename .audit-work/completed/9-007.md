# #9-007 — Critical money-path jobs lack explicit Nightwatch instrumentation

## Plain English

When a payout job fails, the system now explicitly tells Nightwatch about it so the failure shows up as a named, searchable alert — not buried in a generic stream of "queue job failed" notifications. The payout execution job also now tracks how long each payout takes, and flags it with a warning if it's taking longer than expected, giving the team early notice before slowdowns turn into failures.

## Technical Summary

**Files changed:** `app/Jobs/Stripe/ProcessCommissionPayoutsJob.php`, `app/Jobs/Stripe/ExecuteCommissionPayoutJob.php`

- Added `report($e)` in `ProcessCommissionPayoutsJob::failed()` before the existing `Log::error` call. This routes the exception through Laravel's global handler, which Nightwatch hooks to capture named exceptions with full stack context.
- Added `report($e)` in `ExecuteCommissionPayoutJob::failed()` before the existing status-update logic, with the same rationale.
- Added `microtime(true)` timing around `$payoutService->processPayoutBatch($payout)` in `ExecuteCommissionPayoutJob::handle()`. Duration is logged as `duration_ms`. If duration exceeds 30,000ms (25% of the 120s job timeout), a `Log::warning` is emitted with `payout_id`, `duration_ms`, and `attempt` — Nightwatch captures warning-level logs, enabling a dashboard alert on this message. Normal completions emit `Log::info`.

## Decisions Made

- **30s slow-job threshold (not 60s or 90s):** Chosen as 25% of the 120s timeout to give enough headroom for investigation before failures start. At 80% of timeout there's no time to intervene.
- **`$this->attempts()` included in slow-job log:** A batch slow on attempt 4 (Stripe under load) is operationally different from slow on attempt 1; the attempt number informs the response.
- **No config/nightwatch.php publishing:** Nightwatch doesn't support PHP-configurable slow-job alert thresholds; the dashboard alert must be configured there against the `ExecuteCommissionPayoutJob slow payout batch` log message. This is noted in the inline comment.

## Notes

Nightwatch doesn't have a PHP-level config for slow-job alerting (the vendor config only covers sampling, filtering, and ingest settings). The implementation provides the data Nightwatch needs (warning-level log with `duration_ms`) and documents in the comment where the dashboard alert should be wired up.
