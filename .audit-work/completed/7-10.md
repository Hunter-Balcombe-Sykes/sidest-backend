# #7-10 — VideoVariantService.extractPoster writes a 0/1-byte placeholder file on poster-extract failure

## Plain English

When a video's thumbnail couldn't be extracted by FFmpeg (both the 1-second-mark attempt and the 0-second fallback), the code was saving a completely empty file and pretending it was a JPEG image. Anyone visiting a page with that video's thumbnail would see a broken image icon. The fix generates a real (tiny, 1×1 pixel, black) JPEG using PHP's built-in GD image library instead, so the thumbnail slot always contains a valid image file. If GD itself isn't available on the server, the job now fails loudly rather than silently producing garbage.

## Technical Summary

**File changed:** `app/Services/Media/VideoVariantService.php`

- `extractPoster()` (line ~437): Replaced `file_put_contents($output, '')` with a call to new private helper `writePlaceholderJpeg()`. Updated the log message to reflect the new behavior.
- New private method `writePlaceholderJpeg(string $path): bool`: Guards on `imagecreatetruecolor` availability (returns false if GD not loaded), creates a 1×1 true-color image, writes it as JPEG at quality 85 via `imagejpeg()`, and destroys the GD resource. Caller throws `RuntimeException` if this returns false.

The contract of `extractPoster` is unchanged (void return, throws on unrecoverable failure). The poster upload path in `processVariants` is unaffected — it still uploads whatever file lives at `$tmpPoster`, which is now guaranteed to be a valid JPEG.

## Decisions Made

- **Placeholder over hard fail:** Chose a GD-generated placeholder rather than always throwing, to avoid wasting the completed MP4 + HLS encoding work (steps 2–5) when only poster extraction fails at step 6.
- **1×1 pixel size:** Minimal valid dimensions — no need to match the video resolution since this is purely a fallback; clients that render it will scale it anyway.
- **Quality 85:** Standard JPEG quality producing a multi-hundred-byte valid file (not ambiguously small), readable by all decoders.
- **GD unavailable → throw:** If GD isn't present we can't produce a valid placeholder, so failing loudly is correct — better than uploading a corrupt file.

## Notes

No existing tests cover `VideoVariantService` — all 1489 tests passed without modification. The fix is tested implicitly via the service's use of GD, which is a standard PHP 8.2 extension available in the project's runtime environment.
