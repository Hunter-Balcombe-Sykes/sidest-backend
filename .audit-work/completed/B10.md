# B10 — Schedule task `withoutOverlapping`

## Plain English

Two scheduled tasks (`sidest:purge-soft-deletes` and `queue:prune-failed`) were missing the `withoutOverlapping()` guard, meaning they could stack up concurrent runs on multi-server deploys. We added the guard to both.

## Technical Summary

**`routes/console.php` changes:**
- `sidest:purge-soft-deletes`: added `->withoutOverlapping(600)` — 10-hour lock TTL matches the task's nightly cadence and the existing pattern used by other destructive commands (#10-08)
- `queue:prune-failed`: added `->withoutOverlapping()` — default TTL sufficient for a lightweight prune operation (#10-09)

## Decisions Made

- Used `withoutOverlapping(600)` (not the default) for the purge task to match the timeout already applied to the other long-running scheduled commands in the file.
