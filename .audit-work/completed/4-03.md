# #4-03 — `normalizeShopDomain` trim chars include `./` — could mask invalid domains

## Plain English

When a Shopify store URL comes in, the app strips protocol prefixes, ports, and stray dots and slashes from the edges before using it. The problem was that stripping those characters could turn a bad-looking URL (like `../evil.com`) into a clean one (`evil.com`) — and nothing inside the cleanup function itself would catch it. The fix adds a final check at the end of cleanup: if the result isn't a proper `something.myshopify.com` domain, the function returns an empty string, which every caller already treats as "invalid input."

## Technical Summary

**File changed:** `app/Http/Controllers/Concerns/NormalizesShopDomain.php`

Added a `preg_match('/^[a-zA-Z0-9\-]+\.myshopify\.com$/', $value)` guard as the final step of `normalizeShopDomain()`. If the normalized value does not match, the method returns `''`. This is the same pattern already used by `isValidShopDomain()` in `ShopifyAppOAuthController`, but is now enforced at the trait level so all four controllers that `use NormalizesShopDomain` are covered unconditionally — including the `callback()` path, which never called `isValidShopDomain()`.

**New test file:** `tests/Unit/Http/NormalizeShopDomainTest.php` — 9 cases covering happy paths (standard domain, https prefix, trailing slash, leading dot, port stripping) and rejection paths (non-myshopify domain, path-traversal inputs, subdomain spoofing, empty string).

## Decisions Made

- **Guard in the trait, not each caller**: The trait is Shopify-specific by name and contract, so owning the `*.myshopify.com` constraint inside it is the right place. Adds protection to all four callers without per-caller changes.
- **Return `''` on mismatch, not throw**: Consistent with the existing empty-string contract that all callers already handle.
- **Kept `isValidShopDomain` in the controller**: Left as-is since it provides defense-in-depth for the `install()` path and its error message is more descriptive than the generic "missing parameter" error from the empty-string guard.

## Notes

Four controllers use `NormalizesShopDomain`: `ShopifyAppOAuthController`, `ShopifyIntegrationController`, `EmbeddedSetupController`, and `PublicShopifyStorefrontController`. Only `ShopifyAppOAuthController::install()` previously had an explicit `isValidShopDomain` call; the other three callers (including `callback()`) were relying solely on empty-string checks.
