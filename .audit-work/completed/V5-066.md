## Plain English

When a brand's wallet is empty and a top-up arrives in a different currency than the one currently recorded, the system silently switched the wallet's denomination with no record of the change. This is financially significant — it means future payouts would be processed in a new currency — so we now write a log entry every time this happens. The log captures the old currency, the new currency, and which top-up triggered the switch.

## Technical Summary

**Files changed:**
- `supabase/migrations/20260504200000_create_wallet_currency_switch_audit.sql` — new append-only `core.wallet_currency_switch_audit` table (UUID PK, professional_id FK, previous_currency, new_currency, actor_type, actor_id, topup_id, metadata, created_at). Indexed on `(professional_id, created_at desc)`.
- `app/Models/Core/Professional/WalletCurrencySwitchAudit.php` — new Eloquent model following the `ProfessionalDeletionAuditEntry` pattern (HasUuids, no timestamps, booted() sets created_at, constants for ACTOR_TYPE_*).
- `app/Services/Stripe/StripeConnectService.php` — in `applyCommissionTopupFromCheckoutSession()`, the existing currency-change guard flag is now captured as `$currencyChanged`, and after `BrandCommissionTopup::create()` a `WalletCurrencySwitchAudit::create()` is called inside the same `DB::transaction()` block when the flag is true.

**New behavior:** Any wallet currency switch produces one row in `core.wallet_currency_switch_audit` with `actor_type=system`, `trigger=top_up_empty_balance`, and a direct `topup_id` back-reference. If the transaction rolls back, the audit row rolls back too.

## Decisions Made

- Placed the audit write **inside** the transaction so it's atomically consistent with the balance update — a failed top-up cannot produce a phantom audit row.
- Used `actor_type=system` because the currency switch is triggered programmatically by the incoming Stripe checkout session, not by a user action.
- Stored `topup_id` as a direct FK pointer so investigators can join straight to the `BrandCommissionTopup` row that caused the change without needing timestamp matching.
