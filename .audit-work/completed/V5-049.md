## Plain English

Before this fix, the system stopped new platform links from being added once a professional hit their limit of 7, but it never went back and cleaned up accounts that were already over the limit from before that check existed. This added a one-time cleanup command (`sidest:enforce-platform-link-cap`) that finds any professional with more than 7 platform links and soft-deletes the excess ones, keeping the oldest seven. The command has a dry-run mode so you can preview what it would change before running it for real.

## Technical Summary

**Files added:**
- `app/Console/Commands/EnforcePlatformLinkCapCommand.php` — new Artisan command `sidest:enforce-platform-link-cap`. Queries all distinct `professional_id`s with non-deleted `block_group=links` blocks, then for each loads capped-category blocks (social/content/events/streaming) sorted by `created_at ASC`. If count > `platform_links_max` (7), soft-deletes the excess rows via `Block::query()->where('id', ...)->update(['deleted_at' => now()])`. Supports `--dry-run`. Idempotent — already-soft-deleted rows are excluded from the scan. JSON category filtering is done in PHP (not via DB JSON operators) for SQLite/PostgreSQL cross-compatibility.
- `tests/Feature/Console/EnforcePlatformLinkCapCommandTest.php` — 6 Pest tests covering: at-cap (no deletions), over-cap (correct blocks removed), dry-run safety, non-capped category exclusion, idempotency, and per-professional scoping.

**Behavior:** Run `php artisan sidest:enforce-platform-link-cap --dry-run` first to audit existing data, then without `--dry-run` to apply. Logs a start entry with operator and dry-run flag; prints a summary table of professionals scanned/over-cap and blocks soft-deleted.

## Decisions Made

- **Keep oldest, delete newest:** Oldest-first is the safest default — preserves the links the professional intentionally set up first.
- **Soft-delete, not hard-delete:** Consistent with the platform's 30-day retention policy; admin can recover if a professional complains.
- **PHP-side JSON filtering:** Avoids `settings->>'category'` PostgreSQL JSON operator so the command runs identically against the SQLite test DB.
