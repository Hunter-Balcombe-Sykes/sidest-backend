## Plain English

When a brand first connects their Stripe account, the system now automatically sets their commission wallet currency to match their Shopify store's currency instead of defaulting to AUD. This means a USD Shopify store will have a USD wallet from day one, avoiding a mismatch that would have required manual correction before any payouts could be processed.

## Technical Summary

**Files changed:**
- `app/Services/Stripe/StripeConnectService.php` — Added `resolveShopCurrency()` private method that looks up the brand's Shopify integration and reads `provider_metadata['shop_currency']`. In `createConnectAccount()`, the result (if non-null) is merged into the `$update` array so `stripe_manual_balance_currency` is written explicitly at account creation time, overriding the PostgreSQL column default of `'AUD'`.
- `tests/Feature/Stripe/StripeIdempotencyKeysTest.php` — Added `CREATE TABLE IF NOT EXISTS core.professional_integrations` to the `beforeEach` SQLite setup so the new query in `resolveShopCurrency()` resolves without error in the in-memory test environment.

**New behavior:** `createConnectAccount()` queries for a Shopify `ProfessionalIntegration` row for the professional. If found and `shop_currency` is set in `provider_metadata`, that value is normalised to uppercase and written as `stripe_manual_balance_currency`. If no Shopify integration exists (influencers, non-Shopify brands), the column is left unchanged (DB default applies). The existing `?: 'AUD'` fallbacks in the top-up methods remain as last-resort safety nets.

**Contracts:** No API surface change. The field is internal to wallet operations.

## Decisions Made

- Seeding only at `createConnectAccount()` (not `createOnboardingLink()`) keeps the change narrow — reconnecting brands already have a set currency and the existing currency-mismatch guard in `confirmManualTopUpCheckoutSession()` handles drift.
- Chose a direct DB query over a relationship accessor to avoid loading the full integration model unnecessarily.
